집중을 위한 당신만의 미니 웹 도우미

**FocusU**는 집중을 필요로 하는 모든 사람들을 위한 올인원 웹 애플리케이션입니다.  
하루의 할 일 관리부터 감정 일기 작성, 공부 자료 정리, 그리고 AI에게 직접 질문할 수 있는 기능까지 구현했습니다.  
FocusU는 단순하지만 강력한 기능들을 모아 당신의 **집중 루틴**을 정돈해 줍니다.

---

## 🧩 프로젝트 개요

팀: 이찬우(조장), 윤성준, 이승찬이 함께 진행하였으며, 
본 프로젝트는 저희 팀원들의 의견을 바탕으로 제작한 미니 웹 도우미입니다. 
아이폰 캘린더나 공부 사이트 모음집, 홈페이지의 흥미 유발을 위한 타로 카드 기능 등을 기반으로 하였습니다.
아래는 주요 기능들에 대한 간단한 설명입니다.

- 오늘의 집중 키워드를 **타로풍 운세 카드**로 전달합니다.
- 할 일을 정리하고 완료할 수 있는 체크형 **ToDo 리스트**를 제공합니다.
- 하루의 감정과 기록을 담을 수 있는 **다이어리(일기장)** 기능이 있습니다.
- 나만의 공부 자료를 저장하고 관리할 수 있는 **학습 저장소** 기능이 있습니다.
- ChatGPT API 연동을 통해 **AI에게 실시간 질문**도 가능합니다.

## 주요 기능 설명

### 운세(타로) 카드 (Fortune)
- 매일 무작위로 10장 중 5장의 타로풍 카드가 표시됩니다.
- 카드를 한 장 뽑으면, 해당 카드에 어울리는 **오늘의 집중 조언**이 제시됩니다.
- 카드 이미지는 클래식 타로 스타일(GPT로 제작)로 디자인되었으며, 감정적 몰입도를 높입니다.

### 감정 일기장 (Diary)
- 클릭한 날짜에 따라 감정을 입력하고 저장할 수 있는 인터페이스 입니다.
- 사용자만의 **감정 기록**이 가능합니다.

### 할 일 리스트 (Todo)
- 날짜별로 해야 **할 일들을 저장하고 확인할 수 있는 기능**입니다.
- 일기와는 별도로 로컬스토리지에 저장되어, 정보가 섞이지 않습니다.

### 공부 링크 저장소 (Study)
- 자신이 저장하고 싶은 사이트나 링크를 직접 추가 가능하며, 카테고리별로 강의 플랫폼, 공식 문서 등도 기본 제공됩니다.

### AI에게 질문하기 (AI Ask)
- OpenAI API를 연동하면 사용자가 직접 질문을 입력해 ChatGPT로부터 답변을 받을 수 있습니다.
- 현재는 프론트엔드 구조만 제공하며, API 키를 통해 직접 연동해야 합니다. (Vercel 사이트로 진행 가능합니다)
- 추가적으로 본 AI 파일 및 코드는 main.html에 작성되어 있습니다.

**본 프로젝트에 대한 느낀점**

이찬우: 
이번 프로젝트에서 조장을 맡게 된 이찬우입니다. 저의 역할은 read me 작성, 전체적인 페이지 기본 틀 작성 및 branch 나누기(팀원별 일 분배),디자인, 팀원들의 의견 조합을 주로 진행하였습니다. 홈페이지는 단순하게 HTML과 CSS, JS만 잘 다루면 만들어지는 줄 알고 있었습니다. 하지만 본 프로젝트를 진행하며 파일 구조나 디렉러리의 관리 중요성을 알게되었습니다. 특히, 각 기능 별 branch를 main과 병합하는 과정에서 발생하는 충돌과 손대지 말아야 할 부분도 건드리기도 해서 pull/push 전 항상 확인하는 습관이 생겼습니다. 
이런 협업 구조를 경험하면서 다른 버전 관리의 필요성을 체감했고, GitHub를 이용한 실질적인 협업 경험이 저에게 가장 큰 배움이었습니다.

이승찬:
이번 프로젝트에서 투두리스트와 AI기능을 담당하였습니다. 투두 리스트 같은 경우 데이터가 로컬에 남아있도록 처리하는 부분에 많은 신경을 썼고, 아이폰 캘린더 기능을 바탕으로 제작하였습니다. AI 기능에 대하여는 Open AI 기능을 삽입하기 어려워 불가피하게 프론트앤드로 링크만 걸어두게 되었습니다. 사실 저는 이번 프로젝트에서 깃허브 계정은 따로 만들지 않고, 다른 팀원들의 계정을 함께 사용하는 방식으로 작업했습니다. 때문에 브랜치를 따서 작업하기 보다 vs에서 코드를 작성 및 live를 돌려보고 이상이 없으면 다른 팀원들에게 업로드를 부탁하는 방식으로 협업을 했습니다. 그러다 보니 코드 충돌이나 커밋 관리에는 제한이 있었습니다. 그리고 브랜치 전략을 제대로 활용하지 못한 부분이 아쉬웠습니다. 하지만 옆에서 저희 조장이나 팀원이 브랜치를 기능 단위로 따는 과정을 보는 것만으로도 깃허브가 협업에 얼마나 중요한 도구인지 많이 배울 수 있었습니다. 이번 경험을 통해, 혼자 개발하는 것보다 버전 관리와 협업의 흐름을 파악하는 것이 중요하다는 걸 실감했습니다. 다음에는 제 계정을 만들고, 직접 브랜치 전략에 참여해보고 싶습니다.

윤성준:
저는 이번 프로젝트에서 타로카드 운세와 공부 링크 저장소, 감성일기를 담당하였습니다. 타로카드 코드 작성 시 카드 이미지와 운세 문장을 1:1로 매칭시키는 구조 작성에서 큰 어려움이 있었습니다. 조장과 의견 나누기, AI 검색, 구글 검색을 활용하여 1:1 매칭 js 코드를 잘 마무리 할 수 있었습니다. 감성일기 같은 경우 투두리스트와 로컬 저장소를 다르게 하는 점에서 어려움을 겪었으며, key 값 설정으로 해결할 수 있었습니다. 홈페이지를 직접 작성해봄으로써 홈페이지의 구조적인 흐름에 대하여 자세하게 알게 되었습니다. 또한, 깃허브를 통한 협업도 인상 깊었습니다. Branch를 활용해 각자 작업한 내용을 분리하고, 커밋 내역을 남기면서 서로의 작업을 쉽게 확인할 수 있었습니다. 깃허브를 활용한 branch 기반 협업의 효율성을 체감할 수 있었고, 앞으로 더 큰 프로젝트에서도 꼭 활용하고 싶다는 자신감을 얻었습니다.
